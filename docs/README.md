# 文档导航

如果一项信息同时适用于多个主题，只在权威文档中定义，其他文档使用链接引用。

## 维护原则

- 稳定契约与运行快照分离。工具链版本和性能结果随发布证据保存，不复制到多个
  长期文档中。
- 文档描述“为什么”和稳定接口；脚本、workflow、CMake 与测试文件保留易漂移的
  参数和阈值。
- 用户可见行为变化必须同时更新技术规格、变更记录和必要的升级说明。
- 新增文档前先确认现有文档没有合适职责，避免按单次任务创建状态型文档。
